# 教师端学管界面 Skill（由 UI 模式反推）

> **说明**：若当前对话未附带具体截图，本 Skill 以「仪表盘 + 列表 + 输入 + 图表」四类界面为骨架，并结合本仓库 `src/styles/theme.css` 与典型页面实现反推。你后续上传 Figma/截图后，可在「视觉锚点」中补充像素级要求。

## 1. Skill 目标（产品类型推断）

| 维度 | 推断结论 |
|------|----------|
| **主定位** | **教师端学情与教学运营工作台**（诊断 → 排课/规划 → 布置 → 跟进），不是单一的「作文批改」或「学生端练习」工具。 |
| **教研属性** | 公共资源/题库/课程库、排课范围、学习规划分段（已学/未学）等，支撑**教研内容与路径配置**。 |
| **批改/练习** | 界面中**不以逐题批改为核心**；练习/课程多以「列表 + 预览 + 时长」出现，属**任务编排与分发**的一环。 |
| **AI 角色（推断）** | 智能添加任务、诊断链路、反馈收集等——AI 偏**推荐、汇总、辅助决策**，而非替代教师完成全流程批改。 |

**一句话**：让教师在同一套 UI 里完成 **选学生/学科 → 看诊断与规划 → 布置与调整 → 看列表与进度/图表**，必要时提交**文字/截图反馈**。

---

## 2. 输入 / 输出定义

### 2.1 用户输入（界面上「填什么 / 选什么」）

| 区域 | 典型输入 | 用途 |
|------|----------|------|
| **仪表盘 / 顶栏** | 学生、学科、日期/时段、筛选条件 | 限定当前操作上下文 |
| **列表页** | 搜索关键词、下拉筛选、分页、行内操作（复制链接、作废、去排课） | 检索与状态流转 |
| **表单 / 输入框** | 任务说明、备注、时长目标、章节/教材范围、多选标签 | 配置任务与范围 |
| **弹窗 / 侧栏** | 多行文本（字数上下限）、图片附件（张数上限）、必选标签 | 反馈与工单类提交 |
| **图表旁控件** | 时间范围、维度切换（若有） | 驱动看板数据切片 |

### 2.2 系统 / AI 输出（界面上「生成什么 / 展示什么」）

| 输出类型 | 用户可见形态 |
|----------|----------------|
| **结构化列表** | 表格行：学生、学科、章节、状态、创建人、时间等 |
| **统计与进度** | 文案型指标（如「剩余 x/100 次」）+ 环形进度；或「共 n 个任务 / 预计 m 分钟」 |
| **规划/课表视图** | 分章卡片、小节列表（AI 课 / 练习）、已学/未学 Tab、完成/布置标签 |
| **图表看板** | 使用设计 token 中的 `--chart-1` … `--chart-5` 系列色；与主色 `#6574FC` 体系一致 |
| **反馈回执** | Toast/成功文案；错误与校验提示（必选、最少字数等） |

## 3. 核心交互流程（反推逻辑）

以下为主干链路，实现时可拆成独立路由或同一页内多区块。

```text
[上下文选择：学生 + 学科 +（可选）诊断记录]
        ↓
[评估/诊断结果进入] → 列表展示状态（进行中 / 已完成 / 已作废）
        ↓
[排课/规划] → 选教材版本、分册、章节 → 生成或更新计划 → 已学/未学分段展示
        ↓
[布置] → 从规划勾选或智能添加 → 任务清单（可排序）→ 确认发布
        ↓
[看板/列表] → 进度、剩余次数、近期布置摘要（可与图表联动）
        ↓
[反馈]（可选）→ 选类型 + 描述 + 截图 → 提交
```

**要点**：

- **先有上下文，再有动作**：未选学生/学科时，应用空态占位，避免误操作。
- **列表 → 详情/子页**：如「去排课 / 查看排课」从诊断行跳到排课计划页，并携带 `state`（姓名、年级、学科）。
- **强校验步骤**：范围表单（教材/章节）、反馈表单（必选标签、最少字数）在提交前拦截，并用统一 Toast 反馈。
- **图表**通常挂在「汇总层」（今日看板、统计页），与「操作层」（布置、诊断表）通过同一套主色与灰阶保持视觉连续。

## 4. UI 规范摘要（自 `theme.css` 与现有页面归纳）

### 4.1 配色

| Token / 用途 | 色值或变量 |
|--------------|------------|
| 页面/画布背景 | `#F4F7FE`（`--background` / `--dashboard-canvas`） |
| 主色 / 链接 / 强调 | `#6574FC`（`--primary`）、强调文字可用 `#4A4FED`（`--accent`） |
| 主标题灰 | `#444963`（`--page-title-muted`） |
| 正文 | `#101019` 附近（`--foreground` / `--card-foreground`） |
| 次要说明 | `#838BAB`（`--muted-foreground`） |
| 边框 | `#DFE3F0`、`#E9ECF5`、`#CFD5E8`（`--border`、表格分割线） |
| 危险/预警 | `#E0443F`、`#F55656`（错误、剩余次数过低） |
| 图表系列 | `--chart-1` … `--chart-5`（紫/绿/橙/粉等，保持与主色协调） |
| 侧栏选中底 | 浅蓝渐变，如 `from-[#dce8fd] to-[rgba(220,232,253,0.7)]` |

### 4.2 字体与字重

- 基础字号阶梯：`--text-h3` 20px、`--text-lg` 18px、`--text-base` 14px、`--text-sm` 12px。
- 页面主标题：`--font-weight-page-title`（500，与阿里巴巴普惠体 65 Medium 对齐）。
- 列表标题/小节标题：600（`--font-weight-semibold`）；表格/辅助：400（`--font-weight-regular`）。
- 数字指标（如剩余次数）可用 `font-feature-settings: "lnum", "pnum"` 保证等宽数字感。

### 4.3 间距与圆角

- 全局卡片圆角：`--radius` 12px；小组件/标签可用 6px～8px；药丸按钮常用 `rounded-[18px]`。
- 侧栏导航项高度约 **36px**，图标槽 **20×20**，图文间距 **8px**；区块之间纵向 **24px** 分段较常见。
- 主内容区与侧栏：**水平 padding** 常与 `px-6` 级一致；列表卡片 **8px** 栅格间距（如规划双列）。

### 4.4 组件层级

- **一级容器**：白底圆角卡片 + 浅灰描边或投影（弱阴影）。
- **二级行**：列表行 / 任务条高度约 **67px**（布置清单），带左侧拖拽或勾选。
- **弹层**：遮罩 `rgba(16,18,25,0.4)`；右侧抽屉宽约 **520px**，内容区可 `#F5FAFF` 底 + 白卡片分区。

## 5. Agent 使用本 Skill 时的检查清单

实现或还原类似界面时，依次确认：

1. **是否先锁定上下文**（学生/学科/诊断）再展示主功能？
2. **列表**是否具备：表头样式、行高、操作列、空态与加载态？
3. **输入**是否有：占位、边框色、聚焦环（`--ring`）、字数统计与校验提示？
4. **图表**是否使用 `--chart-*` 与主色，而非随意新色？
5. **关键操作**是否有 Toast 与防重复提交（乐观更新可选）？
