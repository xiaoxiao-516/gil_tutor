# 小鹿爱学学管端 Demo 说明

## Demo 意图

这个 Demo 的目标不是还原完整生产系统，而是用一个高保真后台 UI，快速验证「自习室场景学管端」的信息结构和关键工作流是否合理，方便用于：

- 团队内部对齐产品结构和模块边界；
- 与业务方演示“体验 -> 督学 -> 学情 -> 资源”的服务闭环；
- 为后续 PRD、交互稿、前后端开发提供统一参照。

## 产品命名与核心定位

- 产品名：**小鹿爱学学管端**
- 登录方式：账号密码登录（Demo 为非空校验）
- 四大模块：
  1. **体验**：低门槛获客与诊断转化
  2. **督学**：账号创建后的服务落地与效率提升
  3. **学情**：融合校内+自习室数据，支撑续费沟通
  4. **资源**：标准化内容与SOP，保障服务一致性

## 文件结构

```text
study-manager-demo/
  ├─ index.html    # 页面结构（登录页 + 后台主框架 + 四模块面板）
  ├─ styles.css    # 视觉样式（高保真后台风格）
  ├─ app.js        # 交互逻辑（登录、模块切换、风险筛选）
  └─ README.md     # 当前说明文档
```

## 如何运行

1. 直接用浏览器打开 `index.html`；
2. 输入任意非空账号和密码；
3. 进入主后台后，点击左侧导航查看四个模块。

## 当前演示能力

- 登录页与主后台切换；
- 四个核心模块的独立页面信息块；
- 顶部 KPI 指标随模块联动；
- 学情模块中的风险池筛选交互；
- 标准后台组件风格（卡片、表格、状态标签、导航）。

## 后续可扩展方向（建议）

- 对接真实 API 和鉴权（替换 Demo 登录逻辑）；
- 将指标卡与图表替换为实时数据；
- 增加学员详情抽屉、工单流转、消息中心；
- 支持角色权限（学管/督导/组长）视图差异；
- 接入埋点上报（如 `study_task_assigned`、`study_intervention_closed`）。
